기본 상태 목적. 모든 페이지 상단에 노출되는 알림 진입점(벨 아이콘)의 기본 형태를 보여준다. 미확인 알림이 없을 때는 뱃지가 표시되지 않는다.
인터랙션. 벨 아이콘 클릭 시 알림센터 드롭다운(화면 2)이 열린다.
관련 요구사항: NOTI-3 헤더 벨 아이콘 + 뱃지

알림센터 드롭다운 — 목록 (미확인 있음)

기본 상태 · 데이터 있음 목적. 벨 아이콘 클릭 시 열리는 알림센터. 최신순 정렬, 유형(댓글/결제)·요약·발생 시각·읽음 여부를 함께 보여준다. 미확인 개수는 뱃지에 반영된다.
인터랙션. 항목 클릭 → 화면 6(딥링크 이동). "모두 읽음" 클릭 → 화면 5. 목록 하단 "더보기"로 페이지네이션/무한스크롤.
상태 변화. 이벤트 발생 시 3초 이내 이 목록/뱃지 숫자가 실시간으로 갱신되어야 한다(실시간 채널 필요).
관련 요구사항: NOTI-4 알림 목록 UI NOTI-2 실시간 전달 채널 NOTI-3 벨 아이콘 + 뱃지

웹 푸시 권한 요청 프롬프트 (차기 마일스톤)

🕒 차기 마일스톤 (이번 범위 아님) — 웹 푸시는 2026-07-07 변경으로 이번 마일스톤 범위에서 제외되어 연기되었다. 화면은 삭제하지 않고 참고용으로 보존한다.
차기 마일스톤 · 이번 범위 아님 권한 요청 목적. 사용자가 웹 푸시를 활성화하도록 유도하는 브라우저 네이티브 권한 요청 UI(모의). "허용"을 누르면 구독이 등록되고, "차단"을 누르면 인앱 알림만으로 운영된다.
리스크. 사용자가 거부/무시할 수 있으므로 인앱 알림(화면 1~5)이 항상 기본 안전망이 되어야 한다.

웹 푸시 알림 — 앱 미실행 상태 (OS 알림) (차기 마일스톤)

차기 마일스톤 · 이번 범위 아님 앱 미실행 · 백그라운드 목적. 사용자가 앱 탭/창을 닫아둔 상태에서도 이벤트 발생 시 OS 알림 형태로 수신되는 모습을 보여준다.

관련 요구사항(백로그): NOTI-9 웹 푸시 발송 인프라 NOTI-11 Service Worker 푸시 수신/클릭 진입

푸시 미지원 브라우저 안내 (차기 마일스톤)

🕒 차기 마일스톤 (이번 범위 아님) — 웹 푸시는 2026-07-07 변경으로 이번 마일스톤 범위에서 제외되어 연기되었다. 화면은 삭제하지 않고 참고용으로 보존한다.
차기 마일스톤 · 이번 범위 아님 에러/제약 상태 · graceful degradation 목적. 사용자의 브라우저/OS가 표준 Web Push(Service Worker)를 지원하지 않을 때, 왜 푸시 설정이 비활성화되어 있는지 안내하고 인앱 알림은 계속 정상 동작함을 알려준다.
관련 요구사항(백로그): NOTI-12 푸시 미지원 브라우저 안내 UX

10

알림 설정 — 유형별 on/off (인앱 채널)

설정 화면 목적. 사용자가 알림 유형(댓글/결제)별로 인앱 수신 여부를 개별적으로 켜고 끌 수 있다. off로 설정한 유형은 더 이상 인앱으로 발송되지 않는다. 웹 푸시 열은 차기 마일스톤 제공 예정으로 비활성(disabled) 처리되어 있다.
인터랙션. 토글 클릭 시 즉시 저장(자동 저장) 되는 것으로 가정. 웹 푸시 열은 선택 불가 상태이며 "차기 제공 예정" 라벨이 표시된다.
관련 요구사항: NOTI-13 유형별 on/off 설정 UI/API (인앱 채널) NOTI-14 off 설정 유형 발송 차단 (인앱 채널)
